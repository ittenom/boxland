Tasks 63-72: entity types + components + edge sockets + tile groups

Designers can now compose entity types from a catalog of components,
declare project-wide edge-socket types for the Mapmaker's WFC, and build
N×M tile groups for multi-tile brushes.

  Schema (63, 64, 68, 71):
    - entity_types (collider_w/h, anchor_x/y, default_collision_mask, tags)
    - entity_components ((entity_type_id, component_kind) PK)
    - edge_socket_types + tile_edge_assignments
    - tile_groups (1..16 width/height, layout_json as 2D []int64)

  Component-kind registry (65):
    - internal/entities/components: Definition (Kind, Storage, Descriptor,
      Validate, Default, Decode); Registry; Default() seeds Position,
      Velocity, Sprite, Collider
    - Each component declares Descriptor() so the same generic form
      renderer that drives assets also drives component config UIs
    - 11 unit tests covering registry + each built-in component

  EntityType service + Artifact[T] handler (66):
    - Repo[EntityType] for CRUD; Components/SetComponents for the
      composition table (typed validation, full-replace semantics)
    - EntityTypeDraft + Handler implement publish-pipeline contract
      identical to Asset; structured diffs land in publish_diffs
    - 7 integration tests, including round-trip through the publish pipe

  Entity Manager UI (67):
    - views/entities.templ: list + search + duplicate
    - views/entity_detail.templ: top-level form via the generic renderer +
      one Form per attached component + "Add component" picker (filters
      out already-attached kinds)
    - 11 HTTP routes: list/grid/new/create/detail/delete/draft/duplicate
      and per-component add/save/delete
    - data-bx-collider-overlay canvas paints the AABB on top of the sprite
      and live-updates as the form's collider inputs change
      (web/src/entity-manager/collider-overlay.ts, 2 tests)
    - 5 server-side handler tests covering list, create, detail, component
      lifecycle, draft validation

  Edge sockets (69, 70):
    - SocketRepo + CRUD on the entities service
    - SetTileEdges/TileEdges for the per-entity-type assignment
    - /design/sockets list + create + delete page
    - 5 unit tests including ON DELETE SET NULL cascade verification
    - 1 HTTP handler test

  Tile groups (72):
    - Service: CreateTileGroup (zero-init layout), ListTileGroups,
      UpdateTileGroupLayout (dimension-validated), DeleteTileGroup
    - /design/tile-groups list + create + per-group click-to-paint editor
    - boot.js cell-paint handler: assigns the active entity-type id to a
      clicked cell and re-serializes the grid into a hidden form input
    - 4 unit tests (zero-init, dimension bounds, layout dimension
      validation, delete + not-found)

  Test-helper extraction:
    - new internal/persistence/testdb package with TruncateOrder + Reset()
    - assets, entities, designer test files now defer to testdb.Reset
      instead of maintaining three parallel copies of the wipe list
    - Adding a new table to TruncateOrder updates every package's tests
