  Decisions worth flagging:
    * The defer-constraint trigger on players is the cleanest way to
      assert "needs password OR oauth link" since CHECK constraints
      can't span tables. INITIALLY DEFERRED so the OAuth signup tx
      (insert player + insert link) commits cleanly.
    * Player JWT TTL is 15 min on purpose -- the WS connection
      reauths via the long-lived refresh token if needed. Short
      access tokens limit blast radius if one is leaked.
    * GoogleProvider returns nil when its config is empty so the
      gateway can decide whether to register based on a single check
      ("did NewGoogleProvider yield a value?"). Adding Discord/Apple
      follows the same pattern.
    * The dispatcher's "additive" model (HandlePlayer is reachable on
      both realms; HandleDesigner is designer-only) keeps from
      registering most verbs twice while preserving the strict realm
      boundary on the few designer-only opcodes.
    * The rate limiter is intentionally generous (50/s sustained); it's
      anti-bug rather than DDoS protection. Real DDoS lives at the
      Cloudflare layer in front of Railway.
    * Reconnect handling is built around AckTick + AOI.Reset rather
      than a separate "snapshot resend" message. The next tick after
      Reset() rolls every visible chunk back into DirtyChunks, so the
      reconnect path falls naturally out of the per-tick broadcaster
      loop without bespoke code.
